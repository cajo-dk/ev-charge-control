## 0. Feature Request

- REQUEST ID: fr-002
- DATE: 2026.03.14
- STATUS: Released
- RELATED VERSION: 1.2.0
- IMPLEMENTATION BRANCH: fr-002

## 1. Summary

Add execution control so EV Charge Control can start charging at the calculated time instead of only writing a schedule.
This feature introduces a charger control switch and a schedule authorization helper so EVCC can safely execute the current plan when the scheduled start time is reached.

## 2. Problem Statement

EVCC currently calculates an optimal charging window but does not act on it.
Users still need separate automations or manual intervention to start charging, which breaks the purpose of calculating a precise charging start time.

## 3. Scope

### 3.1. Functional Requirements

- Add a required switch entity config for the charger control switch.
- Add a required `input_boolean` config for schedule authorization.
- Check execution eligibility every minute.
- When the calculated start time has been reached and authorization is enabled, turn on the charger switch.
- When EVCC turns on the charger switch, it must also turn off the authorization helper.
- While the charger switch is on, EVCC must not update the schedule.
- Resume schedule updates only when target SoC is reached or finish-by time has been reached.

### 3.2. Non-functional Requirements

- Preserve the existing result payload shape.
- Keep quarter-hour calculation timing unchanged.
- Keep Home Assistant communication inside the existing API client.

### 3.3. Assumptions and Dependencies

- `schedule_authorized_entity` is a Home Assistant `input_boolean`.
- `charger_control_switch_entity` is a Home Assistant `switch`.
- EVCC uses the existing result helper payload as the schedule source of truth.

## 4. Out of Scope

- Forcing the charger switch off at finish-by time.
- Extending the result payload with extra execution metadata.
- Persisting a separate schedule state store outside the result helper.

## 5. Configuration and Data Impact

- Add two new required options:
  - `charger_control_switch_entity`
  - `schedule_authorized_entity`
- No data migration is required.

## 6. Acceptance Criteria

- EVCC can turn on the configured charger switch when a due schedule is authorized.
- EVCC turns off the authorization helper in the same execution path.
- EVCC does not recalculate while the charger switch remains on.
- EVCC resumes recalculation when target SoC is reached or finish-by time is reached.
- Automated tests cover switch control, authorization handling, locking, and due-time evaluation.

## 7. Implementation Notes

- The execution feature should be implemented as a control layer around the existing calculation loop.
- The minute loop should remain the top-level scheduler, with quarter-hour calculations and per-minute execution checks handled separately.

## 8. Test Plan

- Add API client tests for switch turn-on and input_boolean turn-off service calls.
- Add app-level tests for config validation, due schedule execution, authorization blocking, and lock handling.
- Add parsing tests for the result payload time resolution rules.

## 9. Manual QA Checklist

- Configure a valid charger switch and authorization helper in Home Assistant.
- Confirm EVCC starts charging when the schedule becomes due and authorization is on.
- Confirm EVCC turns authorization off when it starts charging.
- Confirm EVCC does not rewrite the schedule while the charger switch remains on.

## 10. Deployment and Rollback Notes

- Configure both new entities in Home Assistant before enabling the feature in production.
- If rollback is required, revert to the previous release and remove the new settings from the add-on configuration.
