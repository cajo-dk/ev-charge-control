## 1. Summary

Implemented the first working EV charging algorithm prototype and scheduled execution loop.

## 2. Issue Description

EVCC could read Home Assistant entity data and write a placeholder result payload, but it did not yet calculate an actual charging window or rerun on a schedule.

As a result, the add-on did not fulfill its primary purpose of determining when charging should start and end based on pricing and finish-by constraints.

## 3. Root Cause

The initial runtime only validated configuration and performed a single placeholder API-backed write. The charging logic, pricing normalization, and recurring scheduler had not yet been implemented.

## 4. Resolution

Implemented the first working calculator in the EVCC runtime. The add-on now:

- normalizes Home Assistant values for SoC, battery capacity, charger speed, charge loss, finish-by, and pricing data;
- evaluates contiguous 15-minute charging windows;
- selects the cheapest valid window that completes before the finish-by deadline;
- writes calculated start and end times to the configured `input_text` helper; and
- runs once at startup and then at minutes `01`, `16`, `31`, and `46` each hour.

## 5. Scope and Impact

This fix changes EVCC runtime behavior, scheduling behavior, and test coverage.

Impact:
- EVCC now produces real charging window output instead of a placeholder payload.
- The add-on recalculates on a fixed recurring schedule after startup.
- Invalid inputs or impossible windows now produce error payloads while keeping the service alive.

Residual risk:
- The prototype uses continuous charging only and rounds required duration up to full 15-minute slots.
- It does not yet support split charging windows or more advanced optimization behavior.

## 6. Validation

Validated by:
- running the automated test suite successfully with `17 passed`;
- adding focused runtime tests for pricing, finish-by handling, no-charge-needed behavior, and no-valid-window behavior;
- adding scheduler tests for the fixed run minutes.

## 7. Deployment Notes

This fix is deployed as version `1.0.3` and is intended to be included in the tagged release contents.

No migration is required, but Home Assistant users must provide valid entity mappings and pricing attributes for the calculator to return `ok`.
