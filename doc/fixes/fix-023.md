## 1. Summary

Corrected EVCC so `Continuous Power` only keeps charger power available after a completed session and does not allow charging to start or continue automatically when scheduling is not authorized.

## 2. Issue Description

EVCC exposed a `Continuous Power` control intended only to keep charger power available after the target SoC had been reached for use cases such as cabin preheating.

In practice, the runtime idle guard treated `Continuous Power` as a general exemption, which meant a de-authorized scheduler could still leave charging active or fail to force the charger back off while the vehicle was requesting charge.

The completion status text also still used the previous timestamp-based wording instead of the required session-complete messages.

## 3. Root Cause

The unauthorized-idle safeguard returned early whenever `continuous_power` was enabled, without checking whether the charge session had actually completed.

Status derivation also evaluated the generic active-charging path before the required completion-state messaging, so a completed session holding continuous power could still appear as an active session.

## 4. Resolution

Adjusted the runtime logic so unauthorized-idle enforcement only allows `Continuous Power` to keep the charger command engaged after target SoC has been reached and the session is complete.

Updated status derivation so completed sessions publish one of the required messages:

- `Charge session complete`
- `Charge session complete - providing continuous power`

Added regression tests covering both the unauthorized continuous-power path and the completion-state status priority.

## 5. Scope and Impact

This fix affects runtime charger-command enforcement and operator-facing status output.

Changed areas:

- `Continuous Power` handling in unauthorized idle states
- charge-session completion status messaging
- runtime regression coverage for completion and unauthorized-start behavior

No configuration changes are required.

## 7. Deployment Notes

After release, EVCC will no longer treat `Continuous Power` as permission to begin charging automatically when scheduling is de-authorized.

When a session completes, the published status message will now clearly distinguish between a fully ended session and a completed session that is intentionally holding charger power available.
