## 1. Summary

Reduced Home Assistant and charger-control traffic by throttling charger-state reads to every 30 seconds and suppressing repeated identical charger switch commands.

## 2. Issue Description

EVCC was polling Home Assistant every second after the immediate-status update change and could reissue the same charger `turn_off` command on each tick while the charger still reported `connected_requesting_charge`.

With the Monta integration behind the selected charger control switch, that caused repeated API calls and led to `429 Too Many Requests` errors.

## 3. Root Cause

The runtime loop was updated to operate on a one-second cadence, but Home Assistant state synchronization and charger-switch service calls still followed that same cadence.

There was no cooldown or deduplication layer to suppress repeated charger switch requests for a desired state that had already just been requested.

## 4. Resolution

Added two guardrails:

- throttled Home Assistant pricing and charger-state reads to once every 30 seconds; and
- deduplicated charger switch commands with a 30-second cooldown for the same requested target state.

EVCC still sends one required `turn_off` when the charger is in `connected_requesting_charge`, but it no longer retries the same off command every second while waiting for the observed charger state to catch up.

## 5. Scope and Impact

This fix affects runtime polling and charger switch command behavior.

Changed areas:

- Home Assistant state synchronization cadence
- charger switch command deduplication
- regression coverage for 30-second read throttling and repeated-off suppression

No configuration changes are required.

After release, EVCC will only refresh pricing and charger-state readings from Home Assistant every 30 seconds.
