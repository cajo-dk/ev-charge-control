## 1. Summary

Added explicit charge-progress logging so EVCC emits `INFO` messages every 15 minutes while charging is active.

## 2. Issue Description

During an active charging session, EVCC continued to publish runtime MQTT payloads on minute ticks, but there were no visible charging-result progress messages in the logs between the initial schedule publish and the final completion publish.
This made it difficult to confirm ongoing charging progress from normal `INFO` logs.

## 3. Root Cause

Runtime output publishing during minute ticks was handled by `write_runtime_output()`, which published MQTT payloads without any matching `INFO`-level progress log.
Only the calculation path emitted an explicit "Published charging result over MQTT" log entry.

## 4. Resolution

Threaded the application logger into the runtime output writer and added a dedicated charge-progress log condition.
The runtime path now emits `Charge progress over MQTT: ...` at `INFO` level only when charging is enabled and the current tick falls on a 15-minute boundary.
Added regression coverage for both quarter-hour and non-quarter-hour charging ticks.

## 5. Scope and Impact

The fix affects runtime logging only.
MQTT publishing cadence and charging control behavior remain unchanged.
The result is better operational visibility during charging without introducing per-minute `INFO` log noise.
