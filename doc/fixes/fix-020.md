## 1. Summary

Changed runtime updates so EVCC reflects state changes immediately, logs state transitions at `INFO` without dumping pricing data, and keeps the charger off when scheduling is de-authorized and continuous power is disabled.

## 2. Issue Description

Three runtime issues remained after `2.2.0`:

- state and activity logging at `INFO` did not consistently reflect control and status changes;
- the charger could still be pulled into charging when the vehicle requested charge on its own while EVCC scheduling was de-authorized; and
- MQTT-visible status changes could wait for the minute loop instead of being published immediately.

That made operator diagnostics weaker and allowed unauthorized charging behavior in a mode that should have been hands-off.

## 3. Root Cause

The runtime still used a one-minute polling loop as its primary execution cadence and only emitted the aggregate progress log on a limited schedule.

At the same time, de-authorized idle behavior relied on the scheduler being disengaged but did not actively force the charger back off when the vehicle moved into a charge-requesting state.

## 4. Resolution

Reduced the runtime poll interval to one second so Home Assistant state changes are reflected back to MQTT without waiting for the next minute boundary.

Replaced the old periodic aggregate progress log with change-based `INFO` logging that records state transitions while intentionally excluding `pricing_information`.

Added an unauthorized-idle safeguard that turns the charger off when scheduling is disabled, manual override is not active, continuous power is off, and the connected vehicle begins requesting charge.

## 5. Scope and Impact

This fix affects runtime control behavior, MQTT update timing, and operator logging.

Changed areas:

- runtime execution loop
- unauthorized charging guard behavior
- `INFO`-level state-change logging
- regression tests for immediate status updates and de-authorized idle behavior

No configuration changes are required.

## 7. Deployment Notes

After release, Home Assistant should see status changes reflected immediately instead of on a one-minute cadence, and `INFO` logs should show state/activity changes without price payload spam.

When scheduling is de-authorized and `Continuous Power` is off, EVCC will now actively keep the charger off if the connected vehicle requests charge on its own.
