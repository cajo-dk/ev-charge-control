## 1. Summary

Corrected the active-charging status text so it shows the expected finish clock time instead of a countdown, and prevented resting no-schedule states from being reported as level `100` system failures.

## 2. Issue Description

Two status presentation problems were present in Release `2.1.0`.

First, the active-charging message used the form `Charge session active - expected finish in hh:mm`, which displayed a remaining duration instead of the expected finish time.

Second, when EVCC was at rest without a calculated schedule and without any actual runtime failure, the placeholder `No schedule calculated.` payload could still cause `status_level` to be published as `100`. That level is reserved for system failures and should not appear during normal idle operation.

## 3. Root Cause

The active-charging message used the computed end-time delta rather than the resolved schedule end clock time.

The false `100` severity came from treating every non-`ok` payload status as a system failure, including the internal no-schedule placeholder used before a schedule exists.

## 4. Resolution

Updated the status derivation logic so:

- active charging now publishes `Charge session active - expected finish at hh:mm`; and
- the internal `No schedule calculated.` placeholder is not treated as a level `100` system failure, allowing resting disconnected states to resolve to the normal `Ready` / level `0` output.

Also added regression tests covering both behaviors.

## 5. Scope and Impact

The fix affects runtime status presentation only.

Changed areas:

- status-message derivation in the EVCC runtime loop
- automated tests covering active charging and idle no-schedule status output

No configuration changes are required.
