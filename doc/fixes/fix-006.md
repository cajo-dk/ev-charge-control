## 1. Summary

Clarified that the changelog must be updated before the release commit is created and corrected the missing `1.0.5` changelog entry.

## 2. Issue Description

The release rules required changelog maintenance, but they did not explicitly state that the changelog update had to happen before the release commit was created. In practice, this allowed a tagged release to exist where the loaded add-on version was newer than the latest changelog entry visible in Home Assistant.

## 3. Root Cause

The governance wording focused on updating the changelog before push or tag, but not specifically before the release commit itself. That left a gap where the tagged commit could still contain stale changelog content.

## 4. Resolution

Updated `CONTEXT.md` and `VERSIONING.md` so `ev_charge_control/CHANGELOG.md` must be updated before the release commit is created. Also added the missing `1.0.5` changelog entry so the changelog now matches the currently released version history.

## 5. Scope and Impact

This fix affects release governance and changelog accuracy only.

Impact:
- Future tagged releases must contain an already-updated changelog.
- Home Assistant should show changelog content that matches the loaded add-on version immediately after upgrade.
- Runtime behavior is unchanged.

Residual risk:
- Older releases created before this clarification may still contain lagging changelog content.

## 6. Validation

Validated by:
- reviewing the updated wording in `CONTEXT.md`;
- reviewing the updated wording in `VERSIONING.md`;
- confirming that `ev_charge_control/CHANGELOG.md` now contains the missing `1.0.5` entry and matches the prior released version.

## 7. Deployment Notes

This fix is deployed as version `1.0.6` and is intended to be included in the tagged release contents.

No migration or Home Assistant configuration changes are required.
