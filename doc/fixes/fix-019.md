## 1. Summary

Corrected status severity derivation so uppercase healthy statuses such as `OK` no longer get misclassified as system failures with level `100`.

## 2. Issue Description

EVCC could publish `status_message=OK` together with `status_level=100` while the system was otherwise at rest and planning normally.

That surfaced a false system-failure severity in Home Assistant even though the underlying state was healthy.

## 3. Root Cause

The status-derivation logic treated only the exact lowercase string `ok` as a healthy calculation status.

When the payload carried uppercase `OK`, EVCC interpreted it as an arbitrary error string and assigned severity level `100`.

## 4. Resolution

Normalized the raw calculation status before evaluating error precedence so healthy status values are handled case-insensitively.

Added a regression test proving uppercase `OK` now falls through to the normal planned-session status logic instead of being treated as a failure.

## 5. Scope and Impact

This fix affects status presentation only.

Changed areas:

- runtime status derivation
- regression tests for case-insensitive healthy status handling

No user configuration changes are required.
