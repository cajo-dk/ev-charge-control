## 1. Summary

Restricted pricing attribute logging so `raw_today`, `raw_tomorrow`, and `forecast` are only emitted at `DEBUG` level.

## 2. Issue Description

EVCC logged the full loaded input structure at `INFO`, which included the full pricing payload. That made normal operational logs unnecessarily noisy and exposed large pricing arrays even when debug-level diagnostics were not requested.

## 3. Root Cause

The runtime logged the entire `LiveInputs` dataclass at `INFO`, and the pricing attributes were part of that object.

## 4. Resolution

Changed the logging behavior so `INFO` now logs only a compact summary of the non-pricing inputs, while the full pricing payload fields `raw_today`, `raw_tomorrow`, and `forecast` are logged only through `DEBUG`.

## 5. Scope and Impact

This fix changes logging behavior only.

Impact:
- Normal logs are shorter and easier to scan.
- Detailed pricing payload inspection is still available when `log_level` is set to `debug`.
- Runtime calculation behavior is unchanged.

Residual risk:
- Users relying on pricing payload visibility at `info` level must switch to `debug`.
