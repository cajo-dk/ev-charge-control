## 1. Summary

Corrected EVCC's MQTT control-state restore path so retained control values such as `target_soc` are restored into the runtime store after reconnects or restarts instead of being ignored or overwritten with blanks.

## 2. Issue Description

Users could see Home Assistant display a valid retained value for controls like `Target SoC` while EVCC still reported runtime errors such as `Missing value for 'target_soc'.`

That mismatch appeared when EVCC restarted or reconnected to MQTT. Home Assistant still knew the last entity state, but EVCC did not reliably restore that state into its internal runtime store.

## 3. Root Cause

EVCC subscribed only to MQTT control command topics and not to the retained control state topics it publishes for Home Assistant.

At the same time, EVCC could publish empty control state values during startup when the internal runtime store was still empty, which risked overwriting retained values before they were restored.

## 4. Resolution

Updated the MQTT integration so EVCC now:

- subscribes to control state topics as well as control command topics;
- restores retained control values from those state topics into the runtime store; and
- skips publishing blank control states when no value is currently known.

Added regression tests covering retained state restoration and blank-state suppression.

## 5. Scope and Impact

This fix affects MQTT control-state synchronization and runtime initialization.

Changed areas:

- MQTT topic subscription and message routing
- control-state publishing behavior
- tests covering retained-state restoration

No user configuration changes are required.
