## 1. Summary

Updated EVCC so the charge-start SoC is captured when the cable is plugged in and remains stable while charging continues.

## 2. Issue Description

The charge-start SoC needed to be refreshed on two concrete runtime events:

- when charging is started by the state machine; and
- when the cable changes from unplugged to plugged.

At the same time, once charging is active, the captured start SoC must not be overwritten by later minute-tick reads.

## 3. Root Cause

The runtime logic already captured `soc_at_charge_start` when the state machine turned charging on, but there was no explicit event-based update for a cable transition from unplugged to plugged.
The value was primarily maintained through fallback resolution logic rather than explicit handling of the required runtime events.

## 4. Resolution

Added explicit detection of the cable transition from `Unplugged` to `Plugged` and set `soc_at_charge_start` to the current SoC when that event occurs.
Kept the existing state-machine capture when charging is turned on.
Added regression coverage to confirm the captured value is not overwritten while charging remains on.

## 5. Scope and Impact

The fix affects only runtime handling of `soc_at_charge_start`.
It improves the accuracy of the recorded starting SoC for dashboards, MQTT consumers, and the optional `input_number` helper.
Charging control behavior is otherwise unchanged.
