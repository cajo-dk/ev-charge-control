## 1. Summary

Ended manual charge sessions automatically when target SoC is reached and `Continuous Power` is off.

## 2. Issue Description

EVCC already stopped scheduler-driven charge sessions at target SoC when `Continuous Power` was disabled.

Manual charge sessions started through `Start / Stop` did not follow the same rule, so charging could remain active after the target SoC was reached even though the user had not enabled continuous power.

## 3. Root Cause

The target-SoC stop rule only existed on the scheduler/state-machine path.

Manual charging bypasses that path, so there was no equivalent guard to end a manual session once the target SoC had been reached.

## 4. Resolution

Added a dedicated manual-session guard in the runtime tick flow.

When a manual charge session is active, target SoC is reached, and `Continuous Power` is off, EVCC now:

- turns the charger off; and
- clears the `Start / Stop` toggle so the session does not immediately restart on the next tick.

## 5. Scope and Impact

This fix affects manual charging behavior only.

Changed areas:

- manual runtime control flow
- regression coverage for manual-session stop-at-target behavior

No configuration changes are required.
