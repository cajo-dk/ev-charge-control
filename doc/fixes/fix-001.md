## 1. Summary

Corrected the repository structure so EVCC is discoverable and deployable as a Home Assistant add-on repository.

## 2. Issue Description

The repository contained add-on deployment files such as `config.yaml`, `Dockerfile`, and `run.sh` at the repository root. Home Assistant add-on repositories are expected to expose a root `repository.yaml` file and one or more add-on subfolders, each containing its own add-on files.

As a result, the repository layout did not match the expected Home Assistant repository format and would not be consumed the same way as the reference add-on repository.

## 3. Root Cause

The initial EVCC scaffold was structured as a single project repository with add-on files placed at the top level, instead of as a Home Assistant add-on repository with a dedicated add-on folder.

## 4. Resolution

Created a root `repository.yaml` file and moved the deployable add-on files into the `ev_charge_control/` folder. Updated the local Python packaging path and README so development from the repo root still works while the add-on remains deployable in the expected Home Assistant structure.

## 5. Scope and Impact

This fix changed repository structure, add-on packaging paths, local packaging configuration, and documentation.

Impact:
- Home Assistant can now consume the repository as an add-on repository.
- Local editable installation continues to work through the updated `pyproject.toml`.
- No charging-algorithm behavior was introduced or changed by this fix.

Residual risk:
- Users or scripts relying on the old root-level add-on file paths must switch to the new `ev_charge_control/` paths.

## 6. Validation

Validated by:
- confirming the repository now contains a root `repository.yaml`;
- confirming the add-on runtime files now exist under `ev_charge_control/`;
- running the automated test suite successfully with `10 passed`;
- pushing the updated structure to GitHub and tagging the release as `v1.0.1`.

## 7. Deployment Notes

This fix is deployed as version `1.0.1` and tag `v1.0.1`.

Consumers of the repository should add or refresh the GitHub repository in Home Assistant after the update so Home Assistant reads the corrected repository structure.
